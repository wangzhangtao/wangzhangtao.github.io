Nexus升级简述

升级原因

由于nexus版本太老，且不支持https镜像同步，所以准备从nexus2升级到nexus3

升级版本规划

Nexus当前版本2.8.1-01

Nexus最新版本3.24.0-02

官网nexus升级 How to Upgrade to Nexus Repository Manager 3.0.x

如果是Nexus 2.x ，需要升级到2.14.1的最新版本 2.14.18 Download Archives - Repository Manager OSS （查看最新2.14的版本）

升级方案

基础环境：java8

版本: 2.8.1-01 –> 2.14.18 –> 3.24.1-02

安装包下载地址： nexus-2.14.18-01-bundle.tar.gz 链接 nexus-3.24.0-02-unix.tar.gz链接

docker镜像下载地址 sonatype/nexus:2.14.18 sonatype/nexus3:3.24.0

nexus版本2.8.1-01升级到2.14.18

官网文档升级步骤

  • Extracting the new release bundle 解压下载的2的最新版
  • Replicating configuration changes 复制配置文件
  • Stopping 2.x instance 停止之前的2版本的进程
  • Replacing the application directory with the new instance 用新的版本作为新的应用程序，即和sonatype-work要在同一目录
  • Starting the new instance 启动新的进程

具体步骤

  1. 解压新的nexus，新的nexu目录必须和sonatype-work在同一层级

tar -xvf nexus-2.14.18-01-bundle.tar.gz

  1. 如果之前的nexus配置文件，即conf下的配置文件做过修改，那么需要覆盖到新的nexus

  2. 停止旧的nexus,正常启动新的nexus即可

nexus-2.14.18-01/bin/nexus start

失败提示：

If you insist running as root, then set the environment variable RUN_AS_USER=root before running this script.

需要你修改nexus-2.14.18-01/bin/nexus，在文件的开始添加RUN_AS_USER=root即可

解压安装包

修改配置文件

启动nexus3

浏览器登录nexus3

http://127.0.0.1:8082/

默认admin密码在文件里

首次登陆后需要修改密码,比如设置为admin123

nexus版本2.14.18升级到3.24.1-02

在nexus2配置升级选项

Click to expand Administration in the left-hand panel. 点击打开左侧的Administration菜单
Click the Capabilities menu item to open the respective screen. 点击Capabilities菜单
Click the New button to access the Create new capability modal. 点击new按钮新建功能模块
Select Upgrade: Agent as your capability Type. 新的功能模块的类型选择Upgrade: Agent，如下图：

通过升级（update）同步数据

此向导将帮助您从Nexus 2升级

添加Nexus2的配置地址和token

您希望同步什么信息

  • 存储库配置和内容

  • 服务器配置信息

仓库数据同步类型

模型有三个选择，Hard link (fastest) Filesystem copy (slow) Download (slowest)

这里由于我们是在同一台机器升级，所以我们选择Hard link (fastest)，这样比较快

选择同步的仓库

查看将要更新的仓库信息

信息更新完毕，继续

开通同步仓库

等待完成以后,直接continue

Any future changes to repositories will not be synchronized. Proceed? 将来对存储库的任何更改都不会同步。继续吗？

意思是后续如果原始库有变动，nexus3新版将不会改变，选择Yes确认。

点击Done,数据同步结束

注意

nexus的路径升级以后有变化

http://*****:8081/nexus/content/groups/public/ 以前的地址

http://*****:8082/repository/public/ 新的地址

这样需要修改maven的 .m2/setting.xml文件, 在打包的时候才能找到正确的依赖包.

如果不这样处理, 我的方法是在仓库前端加一个nginx代理

vim nexos.conf

Rewtite 匹配/nexus/ 跳转到 nexus真正地址/ 参数保持不变，这样就不需要其他人进行修改，保持原有url不变

报错1：Unsupported major.minor version 52.0

错误展示

解决

Unsupported major.minor version 52.0意思是java版本不对，应该是 java-8

请查看java版本号java -version

nexus运行时报错：Connect to sonatype-download.global.ssl.fastly.net:443

处理方法：

​ 登录账号，打开【System】–》【Capabilities】，将【Outreach:Management】禁用即可。

​ Nexus 3 禁用 outreach 功能的方式请参考下面文档：

https://support.sonatype.com/hc/en-us/articles/213464978-How-to-avoid-Could-not-download-page-bundle-messages
